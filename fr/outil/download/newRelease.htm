Cormas 2012 (version Beta)

Présentation

Il s'agit encore d'une version béta qui a été déjà testée, mais qui devra certainement être encore améliorée avant de devenir la version officielle (prévue pour la prochaine école d'été). Les grandes modifications portent sur le coeur de Cormas qui propose une architecture MVC (qui sépare le Modèle de ses Vues et de la manière de le Contrôler). Les développements que nous avons menés, s'orientent vers deux directions : 1) permettre la mise au point de simulations interactives, et 2) faciliter la conception et l'implémentation collective de modèles.

  • Pour le 1er point, il s'agit d'interagir plus facilement avec les entités d'une simulation (les déplacer manuellement, leur envoyer des messages, les pister, zoomer sur des parties de l'espace,...), afin de participer, seul ou à plusieurs, sur le déroulement d'un scénario.
  • Pour le 2ième point, nous cherchons à faciliter l'implémentation d'un SMA. Il s'agit de simplifier la manière de créer des classes, des attributs et leurs valeurs, mais aussi de permettre de dessiner des diagrammes d'activité simples qui soient directement interprétés par Cormas. Il ne s'agit pas de s'abstraire du codage d'un modèle, mais de pouvoir agencer graphiquement des briques d'action entre elles pour produire une nouvelle règle de comportement d'un agent. L'éditeur de diagramme permet de concevoir de nouvelles activités directement executable par Cormas.

Des progrès restent encore à faire dans ces 2 directions. Il s'agit des premiers pas.
Plusieurs autres modifications ont été faites, comme la mise au point d'un module d'analyse de sensibilité ou la création de nouvelles entités (réseaux, arcs et noeuds). Néanmoins, les modèles conçus sous les anciennes versions devraient être compatibles avec ce nouveau Cormas. Nous reprendrons progressivement les didacticiels pour expliquer la manière de concevoir des SMA avec Cormas.
Donc, si vous voulez tester cette nouvelle version, vous ètes les bienvenus ! Envoyer vos retours sur le forum cormas@cirad.fr en indiquant dans le sujet de votre message [beta2012].

Installation

Installer VisualWorks

Cormas a été développé avec la version 7.6 non-commerciale de VisualWorks® (Cincom Systems), qui doit donc être installée sur votre machine. A la condition de vous engager à ne pas utiliser Cormas à des fins commerciales, vous pouvez directement télécharger l'environnement de programmation vw7.6nc (version Windows + Mac OS X) depuis le site de Cormas. Il est conseillé de dézipper l'archive directement sous la racine du disque dur principal de votre ordinateur pour minimiser les problèmes de configuration récalcitrante qui peuvent survenir avec certains systèmes d'exploitation.

Installer Cormas

Concerver l'ancienne version de Cormas

Si une ancienne version de Cormas est déjà installé sur votre machine, vous pouvez garder cette version. Changez simplement le nom du répertoire 'Cormas' en 'Cormas_Old' (ou autre nom).

Installer Cormas2012 pour la première fois

Télécharger la dernière version de Cormas (cormas2012_Full.zip, 13 Mo) qui date de septembre 2012. Dézipper l'archive dans le répertoire VW7.6nc. Le sous-répertoire 'cormas' est créé. Il contient déjà le fichier cormas.im (les instructions en anglais sont détaillées ici). Copiez vos modèles dans le sous-répertoire VW7.6nc/cormas/Models/.

Mettre à jour Cormas2012

Si Cormas2012 est déjà installé, faire une simple une mise à jour en dézippant l'archive cormas2012.zip (1,6 Mo) dans le répertoire VW7.6nc/cormas/. Ouvrir cormas.im et sélectionner le menu '?' et 'Update Cormas'.
