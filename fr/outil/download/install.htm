Cormas (version 2023.04)

Présentation

Dernière version officielle de Cormas (voir les versions antérieures).

Cormas continue son orientation vers deux directions : 1) permettre la mise au point de simulations interactives, et 2) faciliter la conception et l'implémentation collective de modèles. Par rapport à la version précédente, cette nouvelle version met davantage le focus sur le 1er point : interagir plus facilement avec les entités d'une simulation, afin de participer, seul ou à plusieurs, sur le déroulement d'un scénario. Cette version permet donc de :

  • Distribuer une simulation sur plusieurs machines (pour l'instant uniquement sous Windows), et de pouvoir suivre l'évolution à distance d'une simulation, de sélectionner ses propres points de vue et de manipuler les entités.
  • Remonter le temps d'une simulation (backward) ou de rejouer une simulation sauvegardée (replay forward).
  • Piloter des analyses de sensibilité depuis R.

Télécharger Cormas                        

Il s'agit d'une installation minimale de VisualWorks et de Cormas.       

  • Télécharger le fichier cormas2023_package.zip (25 Mo, ou cliquer sur l'image "Download") et dézipper-le sur le répertoire de votre choix.
  • Executer cormas.exe.

Note : cette installation contient le fichier cormas.exe pour Windows. Elle fonctionne aussi sous Mac et Linux, mais dans ce cas il faut ouvrir le fichier cormas.im en l'associant avec cormas/macx/visual.app.

   
Une installation complète de VisualWorks et de Cormas est également possible.

Add-ons: UML et R

Deux nouveaux modules sont disponibles: un éditeur de diagramme de classe UML et des routines R pour piloter Cormas à distance

Editeur UML

Développé par Peter Unhak (Université de Pragues) en tant que module de "OpenPonk" (https://openponk.github.io), cet éditeur permet de dessiner un diagramme de classe (avec attributs, valeurs par défaut, associations, ...) et de générer du code source VisualWorks pour Cormas. OpenPonk est écrit en PHARO, un Smalltalk open-source. Donc pour le moment, cet éditeur n'est pas intégré dans Cormas. Il permet juste de générer des fichiers et de les charger manuellement dans Cormas.

La version actuelle est en version Béta : télécharger openponk.cormas-beta.zip

Routines R 

Pour générer des plans d'expérience et lancer des analyses de sensibilité depuis R, télécharger et dézipper ce fichier (R-Cormas.zip) dans /cormas. Le répertoire contient un document qui explique la procédure.

Si les parcels WebServices ne sont pas sur votre machine lors du chargement de Cormas, télécharger le fichier WebServices.zip et dézipper le dans le répertoire VW7.6/

Dépannage pour Windows 7 et 10

Parfois sous Windows 7, la variable d'environnement VisualWorks ne peut pas être modifiée. Lire la procedure pour régler cela.
